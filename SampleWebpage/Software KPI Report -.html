KPIs in the Report

  • Complexity: Measures the cyclomatic complexity of the code. Lower values are better. High complexity can indicate code that is hard to test or understand.
  • Churn: How often a file changes over time. High churn can indicate unstable or problematic code.
  • Grade: Overall rating (Low, Medium, High) based on complexity and churn. Low is best, High is worst.
  • Hotspot Score: Complexity × Churn. High values indicate files that are both complex and change often—these are hotspots and should be prioritized for review, refactoring, or extra testing.

Interpretation: Aim for low complexity and churn. Files with high complexity and/or churn should be prioritized for refactoring or additional testing.

Hotspot: A file with both high complexity and high churn is a likely source of bugs and maintenance cost. Hotspot analysis helps you find these files quickly.
